# Enterprise AI Platform Documentation

Welcome to the documentation root for the Enterprise AI Platform. This directory contains technical specifications, architectural designs, decision logs, API contracts, and developer onboarding materials.

---

## 📂 Documentation Directory Structure

| Section | Location | Description |
| :--- | :--- | :--- |
| **Architecture** | [`docs/architecture/`](architecture/README.md) | System design, component blueprints, data flows, and security threat models. |
| **Decisions** | [`docs/decisions/`](decisions/README.md) | Architectural Decision Records (ADRs) tracking key engineering decisions. |
| **API Specs** | [`docs/api/`](api/README.md) | REST endpoints, WebSocket protocols, GraphQL schemas, and OpenAPI specifications. |
| **Setup & Onboarding** | [`docs/setup/`](setup/README.md) | Local environment provisioning, prerequisite configuration, and dev workflows. |

---

## 🎯 Documentation Standards

1. **Keep Documentation Up-to-Date**: Any PR modifying system contracts, architecture, or setup steps must include corresponding updates under `docs/`.
2. **Clear Formatting**: Use standard GitHub Flavored Markdown (GFM), Mermaid diagrams for visual workflows, and structured tables.
3. **Architectural Decisions**: Any major tech stack or design decision must be recorded using an Architectural Decision Record (ADR) in `docs/decisions/`.
