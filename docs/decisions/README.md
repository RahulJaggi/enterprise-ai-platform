# Architectural Decision Records (ADRs)

## Purpose

This directory serves as the log for all **Architectural Decision Records (ADRs)** for the Enterprise AI Platform.

An ADR is a document that captures an important architectural decision made along with its context, options considered, trade-offs, and consequences.

---

## 📋 ADR Workflow

1. **Copy Template**: Copy [`docs/architecture/adr-template.md`](../architecture/adr-template.md) to create a new record in this directory.
2. **Naming Standard**: Name the file using the format `NNNN-short-title.md` (e.g. `0001-use-qdrant-vector-database.md`).
3. **Submit for Review**: Submit a Pull Request with status `Proposed`.
4. **Accept & Merge**: Once agreed upon by team deciders, update the status to `Accepted` and merge.
